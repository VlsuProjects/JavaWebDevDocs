# Именованные запросы 

Именованные запросы - это запросы, которые определены в метаданных и могут быть вызваны из кода по имени. То есть вместо того, чтобы писать запрос в коде, вы можете написать его заранее и вызвать его по имени, которые ему присвоите.

Один из способов определения именованных запросов - это использование аннотации `@NamedQuery`. Эта аннотация может быть использована для определения именованных запросов в сущности. Например:

```java
@Entity
@NamedQuery(
    name = "findBookByTitle",
    query = "SELECT b FROM Book b WHERE b.title = :title"
)
public class Book {
    // поля и методы
}
```

В этом примере у нас есть сущность `Book`, в ней мы определяем именованный запрос с именем `findBookByTitle`. Этот запрос выбирает все книги с указанным заголовком.

Обратите внимание, что `b` - это псевдоним для сущности `Book`. Он определяется после названия таблицы в запросе. То, что он
ранее используется после `SELECT`, указывает на то, что мы выбираем все поля сущности `Book`.

Двоеточие перед `title` в запросе указывает на то, что это параметр запроса, который будет передан извне, то есть метод 
`findBookByTitle` должен принимать параметр `title`.
