# Таймеры (Timers)

Таймеры (Timers) служат для генерации пауз (постоянных либо случайных, распределенных по равномерному закону или закону
Гаусса) между отдельными запросами внутри группы потоков. При отсутствии таймера запросы сценария выполняются друг за
другом без задержек, а значит нагрузка, генерируемая одним потоком, будет превышать нагрузку, которую мог бы создать
обычный посетитель.

Таймеры могут быть применены к группе потоков, к отдельному запросу или к обоим вариантам одновременно. В JMeter
представлены следующие типы таймеров:

1. Constant Timer - генерирует постоянную задержку между запросами.
2. Gaussian Random Timer - генерирует случайную задержку между запросами, распределенную по закону Гаусса.
3. Uniform Random Timer - генерирует случайную задержку между запросами, распределенную равномерно.
4. Precise Throughput Timer - генерирует задержку между запросами, чтобы поддерживать заданную пропускную способность.
5. Synchronizing Timer - позволяет синхронизировать выполнение потоков, чтобы они начинали выполнение одновременно.
6. BeanShell Timer - позволяет создать собственный таймер на языке программирования BeanShell.
7. JSR223 Timer - позволяет создать собственный таймер на языке программирования, поддерживаемом JSR223.
8. Constant Throughput Timer - генерирует задержку между запросами, чтобы поддерживать заданную пропускную способность.
9. Poisson Random Timer - генерирует случайную задержку между запросами, распределенную по закону Пуассона.
